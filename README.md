Общая стратегия
---------------

* Учимся по мере возникновения проблем и задач — "решать по мере поступления".
* Параллельно осваиваем связанные технологии (Git, Docker, Redis, Celery, DRF).
* Проекты максимально приближены к реальной работе и тестовым заданиям для собеседований.
* Постепенно переходим от простых приложений к полноценным системам.

Детальный практический план по каждому пункту Django roadmap
============================================================

Уровень 1\. Основы Django и первая практика
-------------------------------------------

### 1\. Установка и настройка Django

* Теория (20%):
    * Быстрый гайд по установке: [Django Official Docs](https://docs.djangoproject.com/en/4.2/topics/install/)
* Практика:
    * Установить Django
    * Создать проект `mysite`
    * Создать приложение `catalog`
    * Запустить сервер и проверить работу
    * Сделать репозиторий в Git
* Задача: Доустановить и развернуть работу простого Django проекта, подготовить его к дальнейшей разработке.

### 2\. Архитектура: MVT (Model-View-Template)

* Теория:
    * Быстрый обзор: [MDN Django Architecture](https://developer.mozilla.org/en-US/docs/Learn/Server-side/Django/Tutorial_local_library_website#architecture_overview)
* Практика:
    * Представить свои понимания архитектуры, нарисовать схему
    * В проекте выделить папки `models`, `views`, `templates`
    * В этом проекте (или новом) реализовать один статичный шаблон
* Задача:
    * Создать пример проекта с новыми моделями, подключить к ним views и шаблоны

### 3\. Создание моделей, миграции, админка

* Теория: [Django Models and Admin](https://docs.djangoproject.com/en/4.2/intro/tutorial02/)
* Практика:
    * Добавить модели `Category` и `Product`
    * Сделать миграции, применить
    * Зарегистрировать их в админке
    * Создать несколько записей через админ-панель
* Задача:
    * Выполнить CRUD операции через админку и через интерфейс сайта

### 4\. Работа с шаблонами

* Теория: [Django Templates](https://developer.mozilla.org/en-US/docs/Learn/Server-side/Django/Templates)
* Практика:
    * Создать базовый шаблон, расширяемые шаблоны
    * Вывести список товаров, реализовать просмотр одного товара
* Задача:
    * Реализовать страницу каталога и страницу товара

### 5\. Авторизация и регистрация

* Теория: [Django Auth](https://docs.djangoproject.com/en/4.2/topics/auth/)
* Практика:
    * Добавить регистрацию пользователя
    * Сделать ограничение доступа к определённым страницам (например, добавление/редактирование товаров)
    * Настроить logout
* Задача:
    * Сделать базовый сайт с регистрацией/авторизацией, доступ к определённым разделам только залогиненным

    * Забронировать/редактировать товар через CBV

* Задача:
    * Реализовать функциональность добавления и редактирования через формы

### 8\. Объектно-ориентированное программирование

* Задача:
    * Внедрить ООП в проект для расширения функционала, создавая свои базовые классы
